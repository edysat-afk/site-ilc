Mapa do Site & Regras de Negócio

Estrutura de páginas e principais regras que guiam o sistema

Hierarquia de Páginas

ilclog.com/
├── Home (/)
├── MASTERMIND 🔒 [membros]
│ ├── infra-2023/ (sessões + PDFs)
│ ├── infra-2024/ (sessões + PDFs)
│ ├── infra-2025/ (sessões + PDFs)
│ ├── infra-2026/ (sessões + PDFs)
│ └── portos/ (sessões + PDFs)
├── servicos/
│ ├── gestao-de-projetos-estrategicos/
│ ├── consultoria-personalizada/
│ ├── treinamento-e-palestras-tecnicas/
│ └── defesa-de-pautas-e-projetos/
├── quem-somos/
├── contato/
├── redes-sociais/
├── login/ + password-reset/
└── category/noticias/ (9 posts)

Regras de Negócio Principais

  • ●
    RN-01 Acesso Mastermind:
    Todo conteúdo das trilhas é restrito. Não autenticado → redirect /login.
  • ●
    RN-02 Serviços sem preço:
    Sem precificação pública. CTA sempre direciona para /contato.
  • ●
    RN-03 Home exibe 3 posts recentes:
    Seção de notícias exibe os 3 artigos com maior data de publicação.
  • ●
    RN-04 Formulário de contato:
    Nome* e E-mail* obrigatórios. Validação client+server. Notifica gerencia.comercial@ilclog.com.
  • ●
    RN-05 PDFs protegidos:
    URLs temporárias (5 min) geradas no servidor. Sem acesso direto sem auth.
  • ●
    RN-06 Roles de usuário:
    admin · editor · member · visitante. Sem self-registration pública.
  • ●
    RN-07 Reset de senha:
    Token enviado por e-mail com validade de 24 horas.

Escolhas técnicas justificadas para a nova plataforma ILC Log

Vantagem da stack escolhida: Next.js 15 com App Router permite renderização híbrida (SSG para páginas públicas, SSR para área de membros), garantindo máxima performance nas páginas indexadas pelo Google e segurança nas páginas protegidas. O Supabase oferece banco PostgreSQL managed com Row Level Security nativo, eliminando a necessidade de servidor dedicado.

Plano de Migração

5 fases de desenvolvimento com entregas e marcos definidos

Fase 1 · Semanas 1–3
Fundação & Setup
Configuração do projeto Next.js + TypeScript, schema Drizzle + migrations PostgreSQL, sistema de autenticação Auth.js, componentes base (shadcn/ui + layout geral).
Next.js setup PostgreSQL migrations Auth.js roles Design system
Fase 2 · Semanas 4–6
Conteúdo Público
Implementação das páginas públicas: Home, Quem Somos, Serviços (4 páginas), Contato com envio de e-mail, Redes Sociais, Blog/Notícias. SEO completo (sitemap, meta tags, og:image).
Home · Serviços Blog Contato + e-mail SEO
Fase 3 · Semanas 7–10
Área de Membros – Mastermind
Área protegida com controle de acesso por role, listagem de trilhas e sessões, player/embed de vídeo, download protegido de PDFs (URLs temporárias de 5 minutos), middleware de autenticação.
Área protegida 5 trilhas PDFs seguros Middleware
Fase 4 · Semanas 11–14
Painel Administrativo
Dashboard com métricas, CRUD completo de posts, serviços, sessões e PDFs, gestão de usuários/membros, caixa de mensagens de contato, editor de conteúdo rico (Tiptap), upload de imagens e arquivos.
Dashboard admin CRUD completo Editor Tiptap Gestão membros
Fase 5 · Semanas 15–16
Polimento, Testes & Go-Live
Testes E2E com Playwright, revisão de performance (Lighthouse ≥ 90), migração de dados do WordPress (export XML), cutover de DNS, monitoramento pós-deploy.
Playwright E2E Lighthouse ≥ 90 Migração WP DNS cutover

Próximos Passos

Ações imediatas necessárias para iniciar o projeto de reconstrução

✦ Ações Imediatas do Cliente

  • 1.
    Validar regras de negócio
    Confirmar ou corrigir as regras levantadas neste documento antes de iniciar o desenvolvimento.
  • 2.
    Definir identidade visual
    Confirmar se o design dos protótipos será adotado ou se há necessidade de revisão de cores, fontes e logo.
  • 3.
    Exportar conteúdo do WordPress
    Realizar export XML do WordPress atual (Ferramentas → Exportar) para preservar posts, páginas e metadados antes da migração.
  • 4.
    Listar todas as sessões do Mastermind
    Catalogar as 120+ sessões com títulos, datas, palestrantes e PDFs para estruturar a migração de conteúdo.
  • 5.
    Confirmar domínio e hosting
    Acesso às credenciais de DNS do domínio ilclog.com para realizar o cutover após o deploy.
